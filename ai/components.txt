COMPONENTS REFERENCE
====================

COMPONENT OVERVIEW
------------------
svelte-docs provides two categories of components:

LAYOUT COMPONENTS:
- DocLayout - Main page layout with navbar, sidebar, footer
- ConfigProvider - Configuration context provider
- Navigation - Sidebar navigation (internal to DocLayout)

CONTENT COMPONENTS:
- ShowcaseSection - Multi-column content sections
- CodeShowcase - Tabbed code examples
- CodeBlock - Single code block with syntax highlighting
- FeatureCard - Feature display card
- CardGrid - Grid layout for cards
- Card - Simple card component
- MermaidChart - Mermaid diagram renderer

All components use Svelte 5 with runes and follow strict naming conventions.

================================================================================
DOCLAYOUT COMPONENT
================================================================================
Main layout component with navbar, sidebar, and footer.

IMPORT:
import { DocLayout } from '@keenmate/svelte-docs';

INTERFACE:
interface Props {

<script lang="ts">

  import { docsConfig } from '$lib/docs-config';

<DocLayout>

  titleText="Getting Started"

</DocLayout>

WITHOUT SIDEBAR:
<DocLayout isNavigationVisible={false}>
  <h1>Landing Page</h1>
  <p>Full-width content without sidebar</p>
</DocLayout>

WITHOUT BREADCRUMBS:
<DocLayout areBreadcrumbsVisible={false}>
  <h1>Clean Page</h1>
  <p>No breadcrumb trail</p>
</DocLayout>

SHOWCASESECTION COMPONENT
================================================================================
Multi-column section for displaying demos, controls, and descriptions.

IMPORT:
import { ShowcaseSection } from '@keenmate/svelte-docs';

INTERFACE:
interface Props {
  titleText: string;                  // Section title
  subtitleText?: string;              // Optional subtitle
  idText?: string;                    // ID for anchor links
  demoContent?: Snippet;              // Demo column content
  controlsContent?: Snippet;          // Controls column content
  descriptionContent?: Snippet;       // Description column content
  col1Title?: string;                 // Column 1 title (default: "Demo")
  col2Title?: string;                 // Column 2 title (default: "Controls")
  col3Title?: string;                 // Column 3 title (default: "Description")
  columnCountType?: 2 | 3;            // Number of columns (default: 3)
  leftColumnSize?: number;            // Left column size (1-12)
  rightColumnSize?: number;           // Right column size (1-12)
  middleColumnSize?: number;          // Middle column size (1-12)
}

3-COLUMN LAYOUT (default):
<ShowcaseSection titleText="Feature Demo">
  {#snippet demoContent()}
    <div class="demo">Live demo here</div>
  {/snippet}

  {#snippet controlsContent()}
    <button>Click me</button>
  {/snippet}

  {#snippet descriptionContent()}
    <p>Description of the feature</p>
  {/snippet}
</ShowcaseSection>

2-COLUMN LAYOUT:
<ShowcaseSection
  titleText="Simple Demo"
  columnCountType={2}
>
  {#snippet demoContent()}
    <div class="demo">Demo</div>
  {/snippet}

  {#snippet controlsContent()}
    <p>Description and controls combined</p>
  {/snippet}
</ShowcaseSection>

CUSTOM COLUMN SIZES:
<ShowcaseSection
  titleText="Custom Layout"
  columnCountType={3}
  leftColumnSize={6}
  middleColumnSize={3}
  rightColumnSize={3}
>
  {#snippet demoContent()}
    <div>Wide demo (6 columns)</div>
  {/snippet}
  {#snippet controlsContent()}
    <div>Controls (3 columns)</div>
  {/snippet}
  {#snippet descriptionContent()}
    <div>Desc (3 columns)</div>
  {/snippet}
</ShowcaseSection>

CUSTOM COLUMN TITLES:
<ShowcaseSection
  titleText="API Example"
  col1Title="Request"
  col2Title="Response"
  col3Title="Notes"
>
  <!-- content -->
</ShowcaseSection>

WITH ANCHOR LINKS:
<ShowcaseSection
  titleText="Feature Name"
  idText="feature-name"
>
  <!-- Link to this: <a href="#feature-name">Jump to feature</a> -->
</ShowcaseSection>

COLUMN SIZE VALIDATION:
- 3-column: left + middle + right must equal 12
- 2-column: left + right must equal 12
- Defaults: 3-col (4-4-4), 2-col (6-6)
- Console warns if sizes don't sum to 12

RESPONSIVE:
- Columns stack on mobile (< lg breakpoint)
- Full width on small screens
- Side-by-side on large screens

COMPONENT PATTERNS
================================================================================

SVELTE 5 RUNES:
All components use Svelte 5 patterns.

$state:
let count = $state(0);
let items = $state<string[]>([]);

$derived.by (ALWAYS use this for functions):
let doubled = $derived.by(() => {
  return count * 2;
});

Access directly in templates:
<p>{doubled}</p>  <!-- NOT {doubled()} -->

$effect:
$effect(() => {
  console.log('Count changed:', count);
});

SNIPPETS:
Children content via snippets:

<ShowcaseSection titleText="Demo">
  {#snippet demoContent()}
    <div>Demo</div>
  {/snippet}
</ShowcaseSection>

NAMING CONVENTION:
All props follow strict naming:
- Boolean: isVisible, hasChildren, canEdit
- Text: titleText, descriptionText, labelText
- Type: variantType, colorType, sizeType
- Collections: navItems, tabItems, buttonList
- URLs: linkHref, websiteUrl, imageUrl

TYPESCRIPT:
All components are fully typed.

import type { DocsConfig, NavigationItem } from '@keenmate/svelte-docs';

RESPONSIVE DESIGN:
All components responsive by default using Bootstrap breakpoints:
- xs: < 576px (mobile)

TROUBLESHOOTING
================================================================================

Component not rendering:
‚Üí Check import path: '@keenmate/svelte-docs'
‚Üí Verify Svelte 5 syntax
‚Üí Check browser console for errors

Types not working:
‚Üí Import types: import type { ... } from '@keenmate/svelte-docs';
‚Üí Verify TypeScript is configured

Styles not applying:
‚Üí Import CSS in layout: import '@keenmate/svelte-docs/styles/main.css';
‚Üí Check Bootstrap classes are correct

Layout broken:
‚Üí Verify config is provided
‚Üí Check navigation structure
‚Üí Ensure Bootstrap grid rules (columns sum to 12)

Code highlighting not working:
‚Üí Verify languageType is correct
‚Üí Check highlight.js supports the language
‚Üí Try auto-detection (omit languageType)

Snippets not working:
‚Üí Use Svelte 5 syntax: {#snippet name()}...{/snippet}
‚Üí Render with {@render snippet()}

Props not reactive:
‚Üí Use $state for reactive variables
‚Üí Use $derived.by for computed values
‚Üí Check prop binding syntax

BEST PRACTICES
================================================================================

1. Component Organization:
   - One component per feature/section
   - Group related showcases together
   - Use semantic HTML inside components

2. Code Examples:
   - Always specify languageType for CodeBlock
   - Use CodeShowcase for multi-file examples
   - Keep code examples concise and focused

3. Layout:
   - Use DocLayout for all pages
   - Disable sidebar only for landing pages
   - Use ShowcaseSection for consistent structure

4. Cards:
   - Use FeatureCard for features/benefits
   - Always wrap in CardGrid for proper spacing
   - Use Bootstrap columns for responsive layout

5. Navigation:
   - Keep menu depth ‚â§ 2 levels
   - Use icons for visual hierarchy
   - Add badges for new features

6. Performance:
   - Lazy load heavy components
   - Keep code examples reasonable size
   - Optimize images before using

7. Accessibility:
   - Provide alt text for images
   - Use semantic headings (h1, h2, h3)
   - Ensure keyboard navigation works

8. Responsive:
   - Test on mobile devices
   - Use Bootstrap breakpoints
   - Stack columns on small screens
